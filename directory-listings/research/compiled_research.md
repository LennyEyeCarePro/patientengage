## PRODUCT FACTS

### What Directory Listings + GBP Overhaul Does:

**Layer 1: Directory Listing Management**
- Pushes verified NAP data to Big Four data aggregators + 50+ authority directories
- Directories include: Healthgrades, Yelp, Bing, manta.com (DA 81), mapquest.com (DA 88), yellowpages.com, superpages.com (DA 74), merchantcircle.com (DA 79), chamberofcommerce.com (DA 60), and more
- Some directories are aggregators that power many downstream sites — fixing upstream cascades corrections downstream
- Practice dashboard shows every listing: status (Live, Pending Approval, Processing, Not Live) + Domain Authority score
- "UPDATE LISTINGS" button pushes current Location data everywhere
- Status donut chart: percentage Live at a glance

**Layer 2: Google Business Profile Overhaul**
- GBP Medical Optimization: category selections beyond generic "Optometrist" to include "Eye Care Center," "Contact Lens Supplier," and specialty categories
- Conversion-First Profile: optimized service descriptions for high-margin specialties (Dry Eye, Myopia, Scleral Lenses) + image metadata for geographic relevance
- Active Engagement Signals: continuous Google Posts and Q&A management for algorithmic "freshness"
- Auto-sync from Location record: toggle on, every save pushes to Google (name, phone, address, hours, description, primary category, website URL)
- Two-way awareness: pulls read-only attributes from Google (accessibility, etc.)

**Layer 3: Facebook Page Management**
- Auto-sync from Location record: name, phone, address, hours, website URL, bio, about section
- Cover Photo management: current + default cover photos, seasonal banner pushes (Glaucoma Awareness, UV Protection, etc.)
- Toggle on/off with last-successful-update timestamp
- Price Range, Email, Intro Bio, About You fields

### How It Works:
1. Practice Location data entered once in PatientEngage
2. Data pushed to 50+ directories via Big Four aggregators
3. GBP connected and optimized with medical categories + specialty descriptions
4. Facebook connected and synced with cover photo management
5. When anything changes: edit Location record once → click UPDATE LISTINGS → all channels update
6. Ongoing GBP management (Posts, Q&A) keeps profile active

### The Single Source of Truth (Location Record):
- Location Name
- Phone Number
- Fax Number (optional)
- Email Contact (optional)
- Street Address 1 + 2
- City, State, ZIP, Country
- Neighborhood
- Location Description (long)
- Short Description
- Tracking Numbers (per channel: Campaigns, GBP, Google Ads)

### 7 Objection Handlers (Dagger Framework)
*Raw Keenan versions in stakeholder_framework.md. Page-friendly versions in step5_competitive_research.md.*

1. "My staff handles listings manually." → Directories auto-overwrite manual changes from third-party data. Without API sync, staff plays Whack-a-Mole. ($25/hr for clerical work a machine does better.)
2. "I already use Thryv/Yext." → Generic tools sync name and address but can't optimize for "Dry Eye Therapy" or "Myopia Management." Data sync is not visibility strategy.
3. "It's not worth the investment." → 10 lost exams/month x ($150 exam + $350 optical) = $5,000/month handed to competitor. Is fixing visibility not worth protecting that?
4. "I'll wait until we update our website." → GBP/directories are the road to the website. A perfect website nobody can find is a luxury showroom in a forest with no roads.
5. "We get enough patients from referrals." → 90% of referred patients still Google you first. Messy GBP = they leak out of the funnel. Referrals are a bonus, not a growth strategy.
6. "We rank #1 for our practice name." → Branded vs non-branded. The real competition is "Best Eye Doctor near me" where 1,200 people in your zip code search and currently find your competitor.
7. "I'm too busy, don't need more patients." → Busy ≠ profitable. Visibility lets you trade low-margin vision plan patients for high-margin medical cases (Dry Eye, Myopia, Sclerals).
